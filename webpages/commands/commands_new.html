  • default command set
  • incubator command set
  • sql command set
assert-content Asserts that the current page contains the expected content.
content The expected content (mandatory).
Wetator exactly asserts the given value by default.
Asserts texts, button labels, alt-texts, title-texts (if visible).

Special features:
  • ' ' - separates two words (more than one space characters are trimmed)
  • ',' - acts as a wildcard for arbitrary words between two expected words
  • '*' - wildcard for arbitrary characters
  • '?' - wildcard for one arbitrary character
  • '[x; y]' - enables table cell addressing
wait The time to wait (in seconds) for the expected content to appear (optional).
example1

Wetator is a tool for automatic Graphical User Interface (GUI) tests of web applications.

command content
assert-content Wetator is a tool

true


command content
assert-content Wetator, tool, web applications

true


command content
assert-content Wet?tor is a t*l

true


command content wait
assert-content Wetator is a tool 5

true if the content "Wetator is a tool" appears at least 5 seconds after the previous step


command content
assert-content Wetator is no tool

false


command content
assert-content tool, Wetator

false (wrong order)
example2

alias favorite domain
goo.gle google.com
smart.testing wetator.org
wet.ator wetator.org

command content
assert-content [favorite domain; wet.ator] wetator.org
assert-content [favorite domain; smart.testing] wetator.org

true since wet.ator and smart.testing both have the favorite domain "wetator.org"


command content
assert-content [favorite domain; goo.gle] wetator.org

false since goo.gle does not have the favorite domain "wetator.org"

open-url Opens a given URL in the current window.
url Specifies the URL to open (mandatory).
May be absolute or relative to the configured base URL (see examples).
examples
command url
open-url http://www.wetator.org

opens www.wetator.org


command url
open-url index.php

opens {wetator.baseUrl}/index.php
