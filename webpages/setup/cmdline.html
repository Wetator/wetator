Command-Line Options

It is possible to start Wetator with

  • a dedicated configuration file and/or
  • one ore many test definition files

Examples:

wetator -p my.properties

Starts wetator with the configuration 'my.properties'

wetator test1.xls

Starts wetator and directly executes test1.xls (no file selection dialog).

wetator test1.xls test2.xls

Starts wetator and directly executes test1.xls and test2.xls.

wetator -p my.properties test1.xls test2.xls

Starts wetator with the configuration 'my.properties'. Wetator directly executes test1.xls and test2.xls.
