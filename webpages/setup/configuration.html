Configuration

wetator.config

The file wetator.config contains the whole configuration for Wetator. You can configure

  • base url (the root for all relative urls)
  • the browser you like to emulate
  • the accept-language Wetator uses (control the language)
  • the directory for the report output
  • as many xml templates you like to generate all reports you (or your boss) needs
  • all the stuff you need to pass your proyx
  • variables, you can use in your tests
  • additional command sets (a way to add your own commands) together with there config

All this options are documented in detail in the configuration file itself. Please have a look at file 'templates\wetator.config'.

Variables

Variables are defined as part of the configuration.
The definition starts always with a '$' sign.
You can refer to this variable from all command parameters with the expression ${var-name}

Example

    $title=My Title
    $app_user=dobby
# Command First Param Second Param
Assert Title ${title}
Assert Content Hello ${app_user}.

 

Sometimes you don't want to have the value of a variable visible in the reports or log files (e.g. passwords). This is possible if you prefix the variable name with '$$'. Then Wetator marks the variable as secrets and outputs '****' instead of the value.

Example

    $app_user=dobby
    $$password=secret

Overwrite properties

It is possible to overwrite all properties. For more see Command-Line Options or Ant Task
