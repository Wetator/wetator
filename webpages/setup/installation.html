Installation & Setup

Prerequisites

  • JDK version 6 or higher
  • If you plan to run Wetator test from ant than you need additionally Ant version 1.7.0 or newer
  • Excel to edit you test plans (future versions will support more formats)

Command Line Version

Installation of the command line version

  • Download the latest stable version.
  • Create a new directory and unzip the file into it (preserve subdirectories).
  • Open a command window and go to your wetator directory.
  • Start wetator [Enter]; If you see some message like 
    Starting Wetator
Wetator execution failed: The configuration file 'D:\Wetator\wetator.config' does not exist.
    then everything seems to be right and you can start with your first test.

Run a simple test

  • Copy the file wetator.config from directory samples\google_search to the root of your installation directory.
    This file defines wetator's setup.
  • Type wetator [Enter]
  • A small popup window appears. Please choose the file google.xls and click on open (You can select more then one file if you like).
  • Wetator executes all commands form the selected file. A small progress report is written to the console.
  • If everything works fine you will end with some output like 
    Tests run: 8,  Failures: 0,  Errors: 0
  • Wetator writes a detailed protocol for every test run. You can find this protocol in the folder 'logs'. Open the file 'SimpleHtml.xsl.html' with your browser.

For more details about the command line interface see Command-Line Options
